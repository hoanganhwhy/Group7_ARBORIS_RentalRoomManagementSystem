Bước 1: Tạo file .env.example
Trong thư mục gốc của dự án, bạn tạo một file tên là .env.example. File này sẽ chứa các biến môi trường cần thiết nhưng xóa bỏ các giá trị nhạy cảm. Bạn có thể để lại thông tin tài khoản ngân hàng hoặc Supabase URL nếu đó là thông tin dùng chung cho cả team, nhưng các key cá nhân thì phải ẩn đi:
# Server Port
PORT=5000

# Supabase Configuration
# Yêu cầu admin cấp quyền hoặc tự tạo project Supabase mới và điền vào đây
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# Bank Account Information (Dùng để nhận thanh toán)
BANK_ID=970422(MB)    (mã đon vị ngân hàng)
BANK_ACCOUNT_NO=Tên tài khoản
BANK_ACCOUNT_NAME="Chủ tài khoản "

# API Keys & Webhooks (Bắt buộc phải điền)
# Lấy API key tại: https://my.sepay.vn/...
SEPAY_WEBHOOK_API_KEY=your_sepay_webhook_api_key

# Lấy API key tại: https://aistudio.google.com/
GEMINI_API_KEY=your_gemini_api_key
Đoạn mã
Lưu ý: Bạn phải đảm bảo file .env (chứa key thật) đã được thêm vào file .gitignore để không bao giờ bị đẩy (push) lên GitHub/GitLab.



Bước 2: Viết hướng dẫn cấu hình (Dành cho file README.md)
Bạn hãy copy đoạn hướng dẫn dưới đây vào file tài liệu hoặc README.md của dự án để gửi cho người khác:
Hướng dẫn cài đặt biến môi trường (Environment Variables)
Để chạy được dự án này trên máy cá nhân, bạn cần thiết lập file .env. Vui lòng làm theo các bước sau:
1. Tạo file .env
Tại thư mục gốc của dự án, sao chép file .env.example và đổi tên thành .env:
•	Trên Windows (CMD/PowerShell): copy .env.example .env
•	Trên Linux/Mac: cp .env.example .env
2. Cấu hình các biến số
Mở file .env vừa tạo và điền các thông tin tương ứng:
•	PORT: Port để chạy server backend (mặc định là 5000).
•	VITE_SUPABASE_URL & VITE_SUPABASE_ANON_KEY: Lấy trong phần Project Settings > API trên trang quản trị Supabase.
•	SEPAY_WEBHOOK_API_KEY: Đăng nhập vào SePay, tạo Webhook và copy API Key dán vào đây.
•	GEMINI_API_KEY: Truy cập Google AI Studio để tạo API Key mới.
•	(Các thông tin tài khoản ngân hàng BANK_* có thể giữ nguyên nếu đang test thanh toán vào tài khoản mặc định).


3. Khởi động dự án
Sau khi đã điền đầy đủ các thông tin trên, hãy lưu file lại và khởi động server (ví dụ: npm run dev hoặc npm start).
